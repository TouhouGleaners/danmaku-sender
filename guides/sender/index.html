    🚀 弹幕发射器 (Sender)

    弹幕发射器是本工具的核心模块，负责将解析后的弹幕高效、平稳地发送至 Bilibili 服务器。v2.3.0 针对大规模补档场景引入了多项智能特性。

    任务准备

    在开始发送任务前，需要进行基础的数据加载：

    1. 视频信息获取（支持智能链接解析）

    在 视频 BV 号 输入框中，你可以手动输入以 BV 开头的 ID，或直接粘贴视频的完整网页链接。

    • 智能解析：支持粘贴如 https://www.bilibili.com/video/BVxxxxxxxxx/ 的链接。
    • 自动定位分 P：如果粘贴的链接带有分 P 参数（例如：.../video/BVxxxxxxxxx?p=3），程序在成功获取信息后会自动为你选中第 3 P，省去在下拉框中手动寻找的麻烦。

    小技巧

    直接从浏览器地址栏复制链接并粘贴，是最快开启任务的方法。

    2. 载入弹幕文件

    点击 [选择文件] 载入本地的 XML 弹幕包。程序会实时解析并显示弹幕总数。

    流程建议

    强烈建议在发送前先进入 [弹幕编辑器] 运行一遍 [开始验证]，清理掉 XML 中的非法换行符和过长内容，避免发送过程中频繁报错，以节约时间。

    核心参数说明

    1. 发送延迟 (Delay Control)

    这是模拟人工操作、防止账号被风控的核心设置：

    • 随机间隔：建议设置为 8.0 - 8.5 秒。太快（如 1-2 秒）极易触发 B 站的 36703: 发送频率过快 限制。
    • 爆发模式：
      • 触发频率：例如每发送 3 条。
      • 休息时间：触发后进入一个长休息（如 30-35 秒）。

    2. 自动终止

    • 已发送条数：达到设定数量后自动停止。
    • 已用时：达到设定分钟数后自动停止。
    • 适用于测试场景。

    v2.3 新特性：任务结束时间预测 (ETA)

    发送器现在能根据你的延迟设置，实时计算剩余任务的期望耗时：

    • 进度展示：进度条会显示当前已尝试发送的数量与总数的百分比。
    • ETA 预测：进度条会显示类似 预计 18:30 结束 的文字。
    • 注：ETA 会根据“随机延迟”和“爆发模式”的动态变化实时修正。

    高级特性

    断点续传 (智能去重)

    勾选 “启用断点续传” 后，程序在发送每条弹幕前会查询本地数据库：

    • 如果发现该视频的同一时间点、同一内容的弹幕已经发送成功，则会自动 跳过 (Skip)。
    • 此功能支持处理 XML 内部的重复弹幕，也支持在程序意外关闭后重新开始任务而不重发。

    运行锁 (Config Lock)

    为了保证任务执行的一致性，当点击 [开始发送] 后，界面上的 BV 号、分 P 选择以及发送策略将被自动锁定（禁用编辑）。

    • 如需修改参数，请先点击 [紧急停止]。

    任务总结与失败处理

    任务结束后，程序会弹窗提示发送结果：

    • 成功：弹幕已进入 B 站待审核池。
    • 智能跳过：已存在于历史记录中的弹幕。
    • 发送失败：
      • 如果存在失败条数，程序会询问是否 “保存失败弹幕”。
      • 建议将其保存为新的 XML，待账号风控解除后重新加载发送。

    桌面通知

    当任务彻底结束或被自动停止时，Windows 任务栏会弹出系统通知，提醒你查看任务报告。
