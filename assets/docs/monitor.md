# 弹幕监视器使用指南

在 v2.1 版本中，监视器已升级为 **数据库审计模式**。它不再依赖模糊的文本比对，而是通过比对 B 站返回的 **唯一身份证 (dmid)**，对您的发送记录进行 100% 精确的存活核销。

### 📊 实时数据仪表盘
新的监视器不再显示进度条，而是通过四个核心指标展示当前视频的弹幕健康状况：

*   **已发送 (Total)**: 本机对当前视频发送成功的弹幕总数（数据来自本地数据库）。
*   **已存活 (Verified)**: <span style="color:green">**绿色**</span>。经系统核实，已成功出现在 B 站在线列表中的弹幕。
*   **待验证 (Pending)**: <span style="color:orange">**橙色**</span>。已发送成功，但尚未在在线列表中扫描到。
    *   *常见原因*: B 站服务器缓存延迟（CDN）、审核队列中。
*   **疑似丢失 (Lost)**: <span style="color:red">**红色**</span>。系统判断可能已被 B 站屏蔽或过滤的弹幕。

### ⚙️ 设置说明
*   **轮询间隔**: 建议 **60 秒**。
    *   这是工具去 B 站“查账”的频率。由于 B 站 XML 列表通常有 3-5 分钟的 CDN 缓存，设置太快并无太大意义。
*   **无需加载文件**: 监视器现在直接读取内部数据库，**不需要**您重新加载 XML 文件，只要在“发射器”页面选定目标视频即可工作。

### 🚀 使用流程
1.  **发送**: 在“发射器”页面正常发送弹幕（发送成功后会自动入库）。
2.  **启动**: 切换到“监视器”页面，确认目标视频无误。
3.  **监视**: 点击 **“开始监视”**。系统将自动轮询并更新仪表盘数据。

### 💡 常见问题与提示

1. “待验证”一直不减少怎么办？
如果超过 30 分钟仍处于“待验证”状态，通常意味着该弹幕触发了 B 站的风控机制，或者进入了人工审核队列。

2. 关于旧版本数据
本功能依赖 v2.1 引入的 `history.db` 数据库。
*   **注意**: 您在 v2.0 及以前版本发送的弹幕，由于当时未记录 `dmid`，**无法**被新版监视器统计到。
